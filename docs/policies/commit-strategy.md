# Commit Strategy

Canonical source for commit behavior.

## Defaults

- Conventional Commits: `type(scope): summary`.
- One commit per completed task by default.
- Keep unrelated changes out of the same commit.

## Recommended types

- `feat`, `fix`, `refactor`, `test`, `docs`, `chore`.

## Commit scope

- Read `TASKS.md` first and align commit scope to active task IDs.
- Include validation notes when non-obvious.
